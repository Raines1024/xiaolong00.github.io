搭建Hadoop-2.9.2集群

准备

在安装 Hadoop 之前，请确认集群的每台机器上均安装 JDK，以及搭配环境变量。

  • 添加Java环境变量
    vim ~/.bash_profile

172.29.32.21 openshift1
172.29.32.22 openshift2
172.29.32.23 openshift3

cd /opt
wget https://mirrors.tuna.tsinghua.edu.cn/apache/hadoop/common/hadoop-2.9.2/hadoop-2.9.2.tar.gz
tar -zxvf hadoop-2.9.2.tar.gz -C .

vim /etc/profile

Flink Environment

export HADOOP_HOME=/opt/hadoop-2.9.2
export PATH=$PATH:${HADOOP_HOME}/bin:${HADOOP_HOME}/sbin

fs.default.name hdfs://openshift1:9000 dfs.replication 3 dfs.namenode.name.dir file:/opt/hadoop-2.9.2/data/namenode dfs.datanode.data.dir file:/opt/hadoop-2.9.2/data/datanode mapreduce.framework.name yarn yarn.nodemanager.aux-services mapreduce_shuffle yarn.nodemanager.aux-services.mapreduce.shuffle.class org.apache.hadoop.mapred.ShuffleHandler yarn.resourcemanager.resource-tracker.address openshift1:8025 yarn.resourcemanager.scheduler.address openshift1:8030 yarn.resourcemanager.address openshift1:8050 openshift2 openshift3 export JAVA_HOME=/usr/java/latest scp -r /opt/hadoop-2.9.2 openshift2:/opt/hadoop-2.9.2 scp -r /opt/hadoop-2.9.2 openshift3:/opt/hadoop-2.9.2 cd /opt/hadoop-2.9.2 hdfs dfs -mkdir /wordcount hdfs dfs -mkdir /wordcount/input # 把当前路径下的 LICENSE.txt 文件复制到 HDFS 中 hadoop fs -put ./LICENSE.txt /wordcount/input # 提交任务，最后两个参数分别指定任务的输入和输出 hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-2.9.2.jar wordcount /wordcount/input /wordcount/output # 查看输出路径 hadoop fs -ls /wordcount/output # 如果一切正常，该路径下包含两个文件 # 第一个文件是空文件，表示任务运行成功 /wordcount/output/_SUCCESS # 第二个文件是输出文件，统计了 LICENSE.txt 中每个单词出现的次数 /wordcount/output/part-r-00000 more VERSION #Mon Sep 02 18:06:26 CST 2019 namespaceID=1033971221 clusterID=CID-98e754ef-ad92-49f8-88b2-6830888f2d48 cTime=0 storageType=NAME_NODE blockpoolID=BP-1400239548-192.168.1.201-1567418785939 layoutVersion=-63 vi VERSION #Sun Sep 01 19:46:01 CST 2019 storageID=DS-330d79ed-7c1b-4d40-b151-81ffcadcf9f0 #clusterID=CID-ae479da3-0b1e-44b0-a383-029a213b3481 clusterID=CID-98e754ef-ad92-49f8-88b2-6830888f2d48 cTime=0 datanodeUuid=67fcc2ae-1b74-46cd-90df-336a0b1950e6 storageType=DATA_NODE layoutVersion=-56 ``` 4）再次启动DataNode，成功启动 sbin/hadoop-daemon.sh start datanode
